Vortex Tech Cyber Security Internship — Week 1

What this is

This repository contains my Week 1 submission for the Vortex Tech Cyber Security Internship Track. The task was to research and document five vulnerabilities from the current OWASP Top 10 list — the industry-standard reference for the most critical web application security risks.

What's inside


vortextech-cybersec-week 1.pdf The main deliverable. For each of five vulnerabilities (Broken Access Control, Security Misconfiguration, Software Supply Chain Failures, Cryptographic Failures, and Injection), it covers:

A plain-language explanation of the vulnerability
How an attacker could exploit it
A real-world example or breach case
Practical prevention/mitigation steps for developers
A closing personal reflection





How to view it

No setup or code execution required — this was a research and documentation task, not a coding one. Just open vortextech-cybersec-week 1.pdf directly on GitHub, or in any Markdown viewer / text editor.

Reference

Research is based on the current OWASP Top 10:2025, the industry-standard list maintained by the Open Web Application Security Project.
